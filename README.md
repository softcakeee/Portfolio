### 스마트팜 거터 운반로봇 개발
-  교수님의 지도 아래 진행된 기업 연계 프로젝트에서 스마트팜 무인 거터 운반 로봇 주행 및 제어 알고리즘 구현을 담당했습니다.

- 참여 인원 수
  - 4명

- 개발 기간
  - 2022.02 ~ 2022.10
  
-  프로젝트 내용
   - 스마트팜 환경에서 양상추 거터를 일정한 간격으로 운반하는 로봇을 개발하고자 함.
   - 로봇이 자신의 위치(구간)를 인식하고 정해진 거리에 맞게 거터를 운반하고자 함.

 -  맡은 역할
    - 소프트웨어 개발 담당
    - 사용언어 : C 및 Python
    - Github : https://github.com/softcakeee/SmartFarm_RGV
    - 로봇 제어 프로그램 개발 (Raspberry PI)

       -  상위 제어 로직 구현
       -  OpenCV OCR 글자 인식을 통하여 로봇의 구간 인식 알고리즘 구현
       -  ATMEGA128과 UART 통신 알고리즘 구현
       -  초음파 센서를 이용한 거리 측정 및 로봇 제어 로직 구현
      
     
    - 펌웨어 개발 (ATMEGA128)

       -  릴레이 제어 모터 구동 알고리즘 구현
       -  Raspberry Pi 와 UART 통신 알고리즘 구현
 
 -  한계점
       -  비전 기반으로 구역을 구분했지만, 농장 환경에서는 시간대와 햇빛 방향에 따라 조도가 크게 달라져 OCR 정확도가 저하되는 문제가 발생했습니다.
       -  초음파 센서로 거터와의 거리를 측정해 위치를 보정했지만, RGV의 이동 속도가 센서 반응 속도보다 빨라 오동작이 발생했습니다.
       -  초기에는 Raspberry Pi가 명령을 전달하고 ATmega128이 모터 제어와 초음파 센서 데이터를 송신하는 구조였으나, 통신 충돌로 인해 엔코더 오류와 송수신 중단 문제가 발생했습니다. 이후 초음파 센서를 Raspberry Pi 쪽으로 옮겨 구조를 단순화했습니다.
         
 -  결과 및 느낀점
    - 처음으로 직접 참여한 프로젝트로, Python과 Linux 환경을 처음 접하며 독학으로 개발을 진행했습니다.
    - 사전 테스트 환경과 실제 농장 환경의 차이가 커서 예상하지 못한 변수가 많이 발생했고, 이에 맞추어 현장에서 코드를 수정하는 경험을 했습니다.
    - 프로젝트 전반의 알고리즘을 직접 구현하며 시행착오를 겪었고, 그 과정에서 문제 해결 능력과 개발 역량을 키울 수 있었습니다.
    - 기업 연계 프로젝트로 마감 기한이 명확했기 때문에 일정 압박을 직접 경험할 수 있었고, 실제 업무 환경의 책임감을 체감했습니다.

- 개발 기간

-  프로젝트 내용
    - 매설물 중 철근 더 나아가 도체를 탐지하는 GPR 같은 역할을 하는 금속탐지기를 개발하고자 함.
    - 코일 전류를 펄스를 발생시켜 자기장을 발생시킬 때 생기는 자기장에 도체가 있을 시 전류의 흐름이 크게 변화하게 되는 것을 이용하고자 함.
    - 탐지 시 도체의 유무를 시각화하고자 함.
    - 도체를 다중 탐지할 시 도체 사이에 거리를 탐지하기 위하여 금속탐지기의 이동거리를 얻기 위하여 홀센서를 이용함.

     
 -  수행 내용
    - 프로그래밍 및 회로 설계/제작
    - 사용언어 : Arduino(C/C++)
    - Github : https://github.com/softcakeee/Metal_Detector_1
    - 펌웨어 개발 (아두이노 나노)
        - 코일을 활용한 회로에서의 전류를 ADC를 사용하여 도체 탐지 알고리즘 개발.
        - 홀센서를 이용한 이동거리 측정 알고리즘 개발.
        - SPI를 사용하는 무선송수신 모듈을 이용한 무선 통신 알고리즘 개발.
        - PLX-DAQ를 활용한 데이터 값 시각화 알고리즘 구현.

 -  동작 결과
    -  코일 테스트
        - 금속 접근 시 LED ON , 부저 울림
        - https://youtu.be/8UTe5IhbuGc
    -  최종 동작
        - 금속 접근 시 부저 울림 및 ADC 값이 떨어지게 됨
        - https://youtu.be/mVNm11L4zWo

 -  한계점
    - 바퀴가 2개인 구조여서 이동할 때마다 균형이 달라졌고, 그로 인해 바닥과 코일 사이의 거리가 일정하지 않아 데이터 값이 불규칙해졌습니다.
    - 단일 코일 구조이기 때문에 금속을 입체적으로 탐지하는 데 한계가 있었습니다.
    - PLX-DAQ를 이용해 Excel로 데이터를 시각화했지만, 실시간 시각화 성능은 충분하지 않았습니다.
        
 -  결과 및 느낀점
    - 코일의 감은 횟수에 따라 감도가 달라져 최적화 과정에서 많은 어려움을 겪었습니다. 이를 통해 센서 감도를 프로젝트 목적에 맞게 조정하는 작업에도 많은 시간과 노력이 필요하다는 점을 배웠습니다.
    - 이동거리 측정을 위해 홀센서를 활용했습니다. 처음에는 홀센서를 단순히 자석 인식용 센서로만 생각했지만, 바퀴 회전 수 계산에 응용할 수 있다는 점을 알게 되었고, 센서의 기본 기능을 어떻게 활용하느냐에 따라 적용 범위가 달라질 수 있다는 점을 배웠습니다.

   - 각 코일에 도체 탐지 여부를 시각화하고자 함.

 -  수행 내용
    - 프로그래밍 및 회로 설계/제작
    - 사용언어 : C 및 Python

       -  pygame 라이브러리를 이용한 게임 패드와 연결 후 데이터 처리 알고리즘 구현.
       -  STM32 와 UART 통신 알고리즘 구현.
       -  matplotlib 라이브러리를 이용한 데이터 값 그래프 시각화 알고리즘 구현.
         
    - 펌웨어 개발 (STM32F103)
        - 여러개의 코일을 활용한 도체 탐지 알고리즘 개발.
        - Raspberry Pi 와 UART 통신 알고리즘 구현.
        - UART 통신을 활용한 조이스틱 모터제어 알고리즘 개발.
        
 -  동작 결과
    -  코일 테스트
        - 금속 접근 시 ADC 값이 감소하며, 그래프에서도 값이 급격히 떨어지는 것을 확인할 수 있음.
        - https://youtu.be/TyNu6CMndYE
    -  최종 동작
        - 금속탐지기가 지나갈 때의 결과 관찰
        - https://youtu.be/BLvZqZDtpuQ

 -  한계점
    - 코일의 크기를 줄이다보니 코일 감도가 많이 낮아져 ADC 샘플링/분해능/속도 등 조정과정을 거쳤지만 만족스러운 결과를 얻어 내지 못하였습니다.
    - 외형은 시중에 파는 RC카를 사용했고 외형 설계의 미스로 홀센서를 설치하지 못하여 1학기때의 이동거리 계산 알고리즘을 구현하지 못하였습니다.
    - STM32 와 Raspberry Pi 사이에 통신에서 서로 송신이 겹치지 않게 하기 위하여 임의의 flag 변수를 만들어 송수신을 원활하게 하도록 설계하였지만 이때 조이스틱값을 실시간으로 보내는 과정에서 실시간으로 그래프를 만들경우 데이터가 송수신량 증가로 반응속도가 느려지는 문제가 발생하였습니다.
      
 -  결과 및 느낀점
    -   학교에서의 마지막 프로젝트라 생각해서 지금까지 학교에서 경험했던 부분들을 한 프로젝트에 담고자 하였습니다.
    -   Raspberry Pi 와 MCU 사이에서의 데이터 교환 부분을 2학년때의 부족했던 부분을 해결하였고 실력이 향상되었다고 느끼는 계기가 되었습니다.
    -   지금까지 사용했던 MCU는 동아리에서 따로 배웠던 것이나 학교에서 배웠던 것을 사용했지만 처음으로 새로운 MCU 를 사용하면서 처음 개발할 때 개발환경 구축과 사용법을 독학하는 경험하게 되었습니다.

### CCTV_PROJECT
-  C++ 학습과 시스템 구조 이해를 위해 진행한 프로젝트입니다.

-  참여 인원 수
  - 1명

- 개발 기간
  - 2025.11 ~ 2026.01
 
-  프로젝트 내용
   - CCTV와 같이 실시간 영상 데이터를 수집하고 처리하는 시스템을 구현하고자 함.
   - 사용자가 웹 브라우저를 통해 실시간 영상 상태를 편리하게 확인할 수 있도록 구성함.
   - Raspberry Pi에 직접 접근하지 않고도 웹 인터페이스를 통해 녹화 기능을 원격 제어할 수 있도록 구현함.
 
 -  수행 내용
    - 프로그래밍
    - 사용언어 : C++
    - Github : https://github.com/softcakeee/CCTV_PROJECT.git
    - Raspberry Pi 기반 소프트웨어 개발
       -  Raspberry Pi 4 기반의 헤드리스 환경에서 동작하도록 구성함.
       -  C++ / OpenCV / cpp-httplib를 사용해 실시간 영상 스트리밍 웹 서버를 구현함.
       -  브라우저에서 /video_feed 경로를 통해 실시간 영상 확인이 가능하도록 구성함.
       -  카메라 처리 루프와 웹 서버를 분리해 영상 수집과 웹 응답이 동시에 동작하도록 구성함.
       -  공유 프레임과 녹화 상태를 다루기 위해 mutex, atomic, condition variable 기반의 동기화 구조를 적용함.

 -  한계점
    - 단순 실시간 모니터링 및 녹화 기능만으로는 사용자가 원하는 장면을 즉시 확인하기 어렵다는 한계가 있었습니다.
    - 녹화 시간이 길어질수록 특정 장면을 다시 찾기 위해 많은 시간을 확인해야 하는 불편함이 있었습니다.
      
 -  결과 및 느낀점
    -   처음으로 C++ 중심으로 진행한 프로젝트였기 때문에 캡슐화와 정보 은닉을 적용하는 과정에서 많은 고민을 했습니다.
    -   구현 과정에서 참조자 사용과 구조 설계 측면에서 아쉬움이 남았고, 객체지향적인 방식으로 코드를 설계하는 데 어려움을 느꼈습니다.
